# Battle card: Скинуться vs Splitwise vs Tricount

Внутренний документ. Цель — честно понять, где мы сильнее конкурентов, где слабее, какие фичи стоит добавить, какие — игнорировать.

> **Замечание о trip-count.com.** Пользователь упомянул `trip-count.com`. Сайт client-rendered, главная страница содержит только лого и заголовок «TripCount — Split trip expenses with friends». По косвенным признакам — клон/SEO-копия известного бельгийского `tricount.com` (Bunq). Сравнение веду с **Tricount**, так как trip-count.com функционально неотличим (или вовсе является его прокси).

## Где мы сильнее (наши УТП)

### 1. «Общий курс проекта» с ретроактивным пересчётом

**Главное отличие.** Ни Splitwise, ни Tricount этого не делают. Конкуренты стампят курс на трату в момент её добавления — навсегда. У нас:

- Курс фиксируется один раз при создании проекта (рыночный).
- В настройках можно поправить — все траты пересчитаются.
- Зачем: в трипе у каждого участника **разный процент конвертации** (1%, 3%, 5%). Договорились о среднем 3% — поставили курс, итоги корректные. У Splitwise придётся либо смириться с распределённой ошибкой по каждой трате, либо перезавести их все.

Это решает реальную боль реальных путешествий, которой никто из конкурентов не закрывает.

### 2. Бесплатная конвертация и автоматический курс

У Splitwise мультивалютная конвертация — Pro ($5/мес). У нас — из коробки. Для пользователя из РФ, для которого Pro-подписка на западный сервис — отдельная боль с оплатой, это бесшовно.

### 3. Публичная ссылка без логина

`/share/<token>` показывает итоговый расчёт незарегистрированному пользователю. У Splitwise/Tricount, чтобы увидеть итоги, нужен аккаунт + приглашение. Хорошо для сценария «отправил ссылку в группу, друзья посмотрели и не парились».

### 4. Никаких лимитов и рекламы

Splitwise в free режиме ограничен 3 тратами в день и показывает рекламу. Tricount показывает мелкую рекламу. У нас всё бесплатно и чисто.

### 5. Русскоязычный UX

Splitwise/Tricount — англоязычные продукты с русским переводом «по остаточному принципу». У нас русский — родной, тон/лексика подобраны под целевую аудиторию (русскоязычные путешественники, в том числе уехавшие).

### 6. Современный стек = быстрые итерации

Next.js 15 + Supabase + Tailwind = одна команда может выкатывать фичу за день. Splitwise/Tricount — большие команды с длинным циклом релизов.

## Где мы слабее

### 1. Только равный сплит между участниками

Нельзя сказать «эта пицца на 60% моя, 40% твоя» или «оплати ровно $50». У обоих конкурентов это есть, у пользователей это часто запрос #1.

**Приоритет фикса:** высокий. Реализовать как минимум «по долям» (1-2-1-2) и «по точной сумме».

### 2. Нет нативных мобильных приложений

PWA работает, но Add to Home Screen на iOS Safari неочевиден; нет push-уведомлений; нет offline-режима полноценного. У Splitwise/Tricount есть iOS + Android.

**Приоритет фикса:** средний. TWA для Android решает половину проблемы быстро. iOS App Store требует Apple Developer Account ($99/год) и больше возни.

### 3. Нет OCR чеков

Splitwise Pro умеет сфотографировать чек и распознать сумму. Это «вау-фича», которая многих привлекает.

**Приоритет фикса:** низкий. Дорого делать, относительно редко используется в действительности.

### 4. Нет повторяющихся расходов

«Каждое первое число месяца — аренда $1000, делим пополам». Есть у Splitwise Pro, нет у Tricount, нет у нас.

**Приоритет фикса:** низкий для трип-кейса (наш фокус), средний для расширения use-case на «соседи по квартире».

### 5. Нет интеграций с платёжками

Кнопка «Заплатить через PayPal» у Splitwise. Технически в РФ это всё равно не работает, так что для нашей аудитории неактуально. Для глобального рынка — слабость.

### 6. Узнаваемость

Splitwise — household name. Tricount — известен в Европе. Нас знает никто кроме нашей группы первых пользователей.

**Приоритет фикса:** маркетинг, не разработка. Лендинг + контент про «как делить трип с разными комиссиями за конвертацию» → сводит к нашему УТП.

## Что добавить в первую очередь, чтобы не уступать по базе

1. Сплит по точным суммам (`по 500₽ с каждого, остальное на меня`).
2. Сплит по долям (`мы вдвоём ели за двоих, дети — за одного`).
3. TWA для Android (упаковка PWA в нативный wrap, 1 день работы).
4. История курсов на трате (мелким шрифтом «по курсу 1₽=X₺») — пользователи путешествующих групп очень хотят прозрачности «откуда взялась сумма».

Эти четыре пункта закрывают 80% запросов, которые могут заставить пользователя уйти на Splitwise/Tricount.
