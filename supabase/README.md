# Supabase

## Текущее состояние MVP

Сейчас используется простая таблица `public.projects` из `schema.sql`:

- `public_id` — публичный идентификатор расчета;
- `name` — название расчета;
- `payload` — JSON с участниками, расходами и настройками сортировки;
- `created_at`, `updated_at` — технические поля.

RLS включен, но политики открыты для `anon`, потому что текущий MVP работает без авторизации и должен открываться по ссылке с разных устройств.

## Следующая миграция

На этапе авторизации нужно добавить миграции в `supabase/migrations`:

- `profiles` — профиль пользователя один к одному с `auth.users`;
- `projects` — проекты пользователя;
- `project_members` — участники проекта с ролями `owner` / `member`;
- `project_invites` — токены приглашений;
- RLS: проекты видят только участники, профиль редактирует только владелец.

Текущий `schema.sql` оставлен для совместимости с уже опубликованным MVP.
